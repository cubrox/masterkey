## 5. Insights and Learnings

### Technical insights

- **HTMX OOB swaps allow one response to update multiple targets.** The questions endpoint returns 3 OOB elements targeting `#question-slot-0/1/2`; comprehension toggle now does the same to sync slots with the button immediately.
- **User colors matter for contrast.** Using `var(--reader-fg)` for the loading placeholder means it automatically contrasts with whatever background the user chose — better than a fixed color.
- **Browser cache is real.** The feature worked server-side immediately, but the browser had to clear cache or hard-refresh before the scattered layout appeared. Live-server mode would've caught this faster.
- **Native `<details>` disclosure works great for question reveal.** No custom JS needed; built-in keyboard support + screen reader announce.

### Process insights

- **Operator gates are async and access-dependent.** The pre-flight verification hit a wall when we didn't have GCP project access, but it's fine to mark gates CONDITIONAL and verify tomorrow — the critical path (magic-link email, health probes, synthetic monitor) all checked out today.
- **Iterative UX refinement works.** Shipped scattered questions, user reported they were all still at the end (turned out to be cache), then found the toggle wasn't syncing — each issue was small and fixable incrementally.
- **Tests catch edge cases, not visual bugs.** 271 tests green; a11y CI caught the contrast violation that unit tests missed. Manual testing on live site found the toggle issue.

### Domain insights

- **M1 is nearly launch-ready.** All code gates pass; only missing human WCAG audit (in flight) and Supabase redirect/RLS verification (tomorrow). The reading surface UX is now polished: consistent width across machines, preferences accessible, and questions thoughtfully placed for reflection.
